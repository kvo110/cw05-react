React CW05 -- Reflection Responses

1. Component Design & Reusability

  Breaking the UI into smaller components made the project easier to build, read, and debug. Instead of putting everything into one large App file, I separated each part into HelloWorld, Counter, List, and FilteredList. Each component has one clear responsibility. For example, the Counter only handles updating the count, while the List only displays items passed in through props. This improves reusability because I can reuse these components in other parts of the app without rewriting them. It also helps with debugging since if something is wrong with filtering, I know to check the FilteredList component instead of searching through the entire project.

2. State vs Props in UI Management

  Props and state are both used to manage data in React, but they serve different purposes. Props are passed from a parent component to a child component and cannot be changed inside the child. State is data that belongs to a component and can change over time. In my project, the Counter component uses state because the count updates whenever the button is clicked. Using setState() is important because it tells React to re-render the UI. If I changed the value directly, React would not detect the update properly. Overall, props are for passing data down, while state is used for data that needs to change inside a component.

3. Asset Organization & Scalability

  If this project expanded to around 50 components, I would organize the files into more structured folders. Right now everything is inside a components folder, but I would split it into folders like components, pages, styles, and possibly utils for helper functions. I would also keep consistent naming like Counter.jsx and FilteredList.jsx so it is easier to find files. Shared styles could be placed in separate CSS files, and images could go into an assets folder. This type of organization would make the project easier to scale and maintain over time.

4. DOM vs React’s Virtual DOM

  React uses a Virtual DOM to improve performance compared to directly manipulating the real DOM. In regular JavaScript, you often have to manually update elements, which can become repetitive and error-prone. In React, when state changes, React compares the new Virtual DOM to the previous one and only updates the parts that actually changed. For example, in the Counter component, only the number updates when the button is clicked instead of re-rendering everything. This makes the code easier to understand and improves efficiency.

5. UI/UX Decisions for Learning & Accessibility

  In the FilteredList component, both usability and accessibility are important. To improve accessibility, labels could be added to the search input and dropdown so screen readers can describe them properly. Keyboard navigation should also work so users can interact without a mouse. From a UI perspective, grouping the search bar and dropdown together makes it easier to understand how filtering works. Clear headings and consistent styling also help users quickly understand each section of the app. Overall, improving accessibility also improves the experience for all users, not just those with disabilities.
